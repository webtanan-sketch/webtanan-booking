# فاز اول: مستندسازی و پایه فنی

هدف فاز اول این است که پروژه قبل از اضافه شدن جزئیات سنگین محصول، یک ستون فقرات واضح داشته باشد. این فاز شامل دو خروجی همزمان است:

1. مستندات قابل تحویل به تیم فنی
2. افزونه پایه قابل نصب در وردپرس

## محدوده فاز اول

- تعریف معماری Hybrid
- ساخت CPT پزشکان
- ساخت جدول‌های اختصاصی اصلی
- تعریف نقش‌ها و capabilityها
- ثبت APIهای پایه
- پیاده‌سازی lock نوبت با transaction
- ثبت دفتر کل کیف پول
- ثبت OTP هش‌شده
- اسکلت SMS و payment adapter
- شورت‌کدها و Elementor wrapperها

## خارج از محدوده فاز اول

- اتصال واقعی به درگاه پرداخت
- verify واقعی تراکنش بانکی
- اتصال production به IPPanel
- تقویم شمسی کامل با UI روزانه/هفتگی/ماهانه
- داشبورد کامل پزشک و منشی
- فرآیند کامل تسویه تایید/رد/پرداخت
- تست بار همزمانی روی سرور مقصد

## تعریف Done برای فاز اول

- افزونه بدون fatal error نصب شود.
- جدول‌ها در activation ساخته شوند.
- یک پزشک از پیشخوان قابل ایجاد باشد.
- برنامه کاری از REST داشبورد قابل ثبت باشد.
- اسلات‌ها از REST خوانده شوند.
- یک اسلات با lock ۱۵ دقیقه‌ای قفل شود.
- lock منقضی‌شده در کران expired شود.
- OTP خام در دیتابیس ذخیره نشود.
- مستندات در پوشه `docs` همراه افزونه باشد.
