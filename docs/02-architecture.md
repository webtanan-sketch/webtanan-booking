# معماری

## مدل Hybrid

افزونه از دو بخش اصلی تشکیل شده است:

## 1. محتوای عمومی با WordPress CPT

پروفایل پزشک با Custom Post Type به نام `saas_doctors` ساخته می‌شود.

کاربردها:

- صفحه اختصاصی پزشک
- آرشیو پزشکان
- سازگاری با Elementor Theme Builder
- سازگاری با افزونه‌های SEO مثل Rank Math و Yoast
- ایندکس شدن صفحه پزشک در گوگل

## 2. عملیات پرتراکنش با Custom Tables

داده‌های مالی، رزرو و عملیاتی در جدول‌های اختصاصی نگهداری می‌شوند.

نمونه‌ها:

- `wp_saas_appointments`
- `wp_saas_transactions`
- `wp_saas_wallets_ledger`
- `wp_saas_schedules`
- `wp_saas_otp_logs`
- `wp_saas_sms_logs`

## مرز مسئولیت‌ها

اطلاعات SEO، متن معرفی، تصویر و محتوای عمومی پزشک در `wp_posts` است.

اطلاعاتی مثل تعرفه، وضعیت تایید، شماره نظام پزشکی، تنظیمات پرداخت، برنامه کاری و موجودی مالی در جدول‌های اختصاصی است.

## کش

صفحه پزشک می‌تواند توسط افزونه‌های کش ذخیره شود. اما این داده‌ها نباید داخل HTML اولیه ثابت شوند:

- اولین نوبت خالی
- اسلات‌های آزاد
- وضعیت locked بودن اسلات
- موجودی کیف پول
- وضعیت پرداخت
- رسید پرداخت

تمام این موارد باید از REST API خوانده شوند.

## Provider Adapterها

اتصال providerها به صورت لایه جداگانه نگه داشته می‌شود. پیامک با IPPanel و پرداخت با آقای پرداخت پیاده‌سازی داخلی دارند، اما ساختار همچنان قابل توسعه است.

- `webtanan_booking_available_gateways`
- `webtanan_booking_gateway_adapter`
- `webtanan_booking_sms_send`

این تصمیم باعث می‌شود اطلاعات حساس در تنظیمات ذخیره شود و هر سایت بتواند providerهای بعدی را بدون تغییر هسته افزونه وصل کند.
