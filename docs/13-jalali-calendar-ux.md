# نسخه 0.7.0: تقویم شمسی و تجربه رزرو

در این مرحله اولین لایه تقویم شمسی واقعی افزونه اضافه شد. هدف این تغییر این است که کاربر فارسی‌زبان در صفحه پزشک، داشبورد مطب و بخش‌های مدیریتی با `input date` خام مرورگر روبه‌رو نشود، اما قرارداد دیتابیس و REST همچنان با تاریخ استاندارد `YYYY-MM-DD` میلادی باقی بماند.

## فایل‌های اصلی

- `assets/js/jalali-calendar.js`
- `assets/js/frontend.js`
- `assets/css/frontend.css`
- `assets/css/admin.css`
- `includes/class-frontend.php`
- `includes/class-admin.php`

## اصول پیاده‌سازی

- تقویم شمسی به صورت ماژول JS داخلی و سبک پیاده‌سازی شده است.
- هیچ کتابخانه خارجی سنگین به افزونه اضافه نشده است.
- مقدار اصلی فیلد تاریخ برای فرم‌ها و API همان Gregorian `YYYY-MM-DD` است.
- نمایش کاربر، عنوان ماه، روزها و انتخابگر تاریخ به فارسی و RTL است.
- inputهای تاریخ داخل `.webtanan-booking` و `.webtanan-admin` به صورت خودکار به انتخابگر شمسی تبدیل می‌شوند.

## محل‌های استفاده

- تقویم رزرو در صفحه عمومی پزشک
- داشبورد پزشک و منشی
- فرم ثبت نوبت حضوری
- فرم مرخصی و استثنائات
- فیلترهای تاریخ در مدیریت

## داشبورد پزشک و منشی

نمای تقویم داشبورد مطب اکنون سه حالت دارد:

- روزانه
- هفتگی
- ماهانه

برای حالت هفتگی و ماهانه، کلاینت بازه تاریخ را تولید می‌کند و برای هر روز داده را از REST می‌گیرد. بنابراین اسلات‌ها و وضعیت‌ها همچنان در HTML اولیه cache نمی‌شوند.

## وضعیت‌ها

وضعیت‌های زیر با برچسب و رنگ‌بندی فارسی در UI نمایش داده می‌شوند:

- آزاد
- در حال رزرو
- رزروشده
- پرداخت در مطب
- مراجعه کرد
- مراجعه نکرد
- لغوشده
- منقضی

## محدودیت باقی‌مانده

این نسخه فاز اول تقویم شمسی است. برای نسخه‌های بعدی باید موارد زیر کامل‌تر شوند:

- Drag/resize رویدادها در تقویم مطب
- مشاهده جزئیات نوبت از داخل کارت تقویم
- فیلتر وضعیت‌ها داخل نمای تقویم
- تست مرورگری کامل روی موبایل، تبلت و Elementor
